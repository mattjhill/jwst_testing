Tests¶

Descriptions of all tests and their source code can be found below. Tests are organized into classes with each step of the pipeline having its own class (e.g. TestDQInitStep contains all of the tests for Data Quality Initialization.)

class stvalidate.test_pipeline.TestResetStep[source]¶

The base class for testing the Reset-Anomaly Correction.

test_reset_correction(input_file, refhdu, output_file, request, chapter)[source]¶

Check that for each integration in the input science data, the reset corrections are subtracted, group by group, integration by integration. If the input science data contains more groups than the reset correction, then the correction for subsequent groups is zero. If the input science data contains more integrations than the reset correction, then the correction corresponding to the final integration in the reset file is used. Only performed for MIRI data.

class stvalidate.test_pipeline.TestLinearityStep[source]¶

The base class for testing the Linearity.

test_linearity_correction(output_file, refhdu, input_file, request, chapter)[source]¶

Check that the linearity correction is properly applied to all relevant pixels. The algorithm uses a polynomial of the form

\[F_c = \sum_{i=0}^N C_i F^i\]

where \(F_c\) is the corrected counts, \(C\) are the correction coefficients, and \(F\) is the uncorrected counts. The coefficients of the polynomial at each pixel are given by the reference file.

test_linearity_median_residuals_rms_lt_1percent(_percent_rms, request, chapter)[source]¶

Check that after the linearity correction the ramps agree with a linear fit to an percent RMS of less than 1%. Where

\[\%RMS = \frac{F - F_c}{\max{F_c}} * 100\]
test_linearity_99percent_of_residuals_rms_lt1percent(_percent_rms, request, chapter)[source]¶

Check that more than 99% of pixels’ residuals have percent RMS < 1%.

test_linearity_pixeldq_propagation(output_file, refhdu, input_file, request, chapter)[source]¶

Check that all DQ flags are propagated from the reference file (header keyword R_LINEAR) to the output PIXELDQ array.

Supported Flags¶

0 1 DO_NOT_USE Bad pixel.
1 2 NONLINEAR Pixel highly nonlinear.

class stvalidate.test_pipeline.TestDarkCurrentStep[source]¶

The base class for testing the Dark Current Subtraction.

test_dark_current_subtraction(output_file, refhdu, input_file, request, chapter)[source]¶

The Dark Current step can use reference files with one of two formats: If the reference file contains a ramp with NFRAMES=1 and GROUPGAP=0, then the program averages and skips dark frames to match the NFRAMES and GROUPGAP values of the science data, before performing a group-by-group subtraction of the dark-reference ramp from the science data. To speed processing, if the reference file has NFRAMES and GROUPGAP values that match those of the data file, then the ramp will be subtracted as is, without additional averaging. If the dark ramp has more groups than the science image, the extra dark groups are ignored. If the dark ramp has fewer groups than the science image, the Dark Current step will issue a warning and return the science image unchanged.

test_dark_current_pixeldq_propagation(output_file, refhdu, input_file, request, chapter)[source]¶

Check that all DQ flags are propagated from the reference file (header keyword R_DARK) to the output PIXELDQ array.

Supported Flags¶
Bit Value Name Description
0 1 DO_NOT_USE Bad pixel.
1 2 HOT Hot Pixel.
2 4 WARM Warm pixel
3 8 UNRELIABLE_DARK Dark variance large
4 16 UNRELIABLE_SLOPE Slope variance large
